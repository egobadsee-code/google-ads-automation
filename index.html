Google Ads API Integration

How Augis uses the
Google Ads API

Augis is a digital marketing agency that builds and operates an in-house Google Ads API tool to create, manage and report on Search and Performance Max campaigns — for our own advertiser accounts and for client accounts we manage under explicit client authorization.

Who Uses the Tool

Augis is an advertising agency. The tool manages our own Google Ads accounts and our clients' accounts (mainly dental and healthcare clinics in Brazil). Every client account is linked to our manager account (MCC 941-449-3856) only after the client explicitly grants access. The tool is internal to Augis and is not distributed, licensed, or resold.

Technical Stack

Built with Python, the official google-ads Python SDK, a FastMCP server, and OAuth2 installed-app flow. Runs locally on the agency operator's machine. Authentication uses a refresh token stored in a local encrypted environment file. No cloud deployment, no third-party hosting.

Manual Confirmation Gate

All write operations (campaign creation, budget changes, bid and keyword adjustments) require explicit manual confirmation by the operator before any API call runs. Read and reporting operations run automatically. No fully automated, unattended account changes.

API Services Used

Reporting: GoogleAdsService.Search / SearchStream for campaign, budget, ad group and keyword performance. Management (with confirmation): MutateCampaigns, MutateCampaignBudgets, MutateAdGroups, MutateAds, MutateAdGroupCriteria. Categories: Campaign Management, Reporting, Keyword Planning.

Data & Access Scope

Operated by the agency owner (egobadsee@gmail.com) under MCC AugiScale (941-449-3856). Expected volume: under 1,000 API calls/day across a small number of own and client accounts. Performance data is used only to manage the linked accounts and is never sold or shared with unrelated third parties.

Compliance

This integration complies with the Google Ads API Terms of Service and Google's advertising policies. Client accounts are accessed only with the client's authorization through standard MCC linking, and access is removed when an engagement ends.
